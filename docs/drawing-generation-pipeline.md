# Drawing Generation Pipeline

## Goal

Convert a confirmed BridgeModel into deterministic drawing instructions that can
be mapped to AutoCAD entities.

## Pipeline

```text
BridgeModel
  -> DrawingPlan
  -> CadInstruction[]
  -> AutoCAD Entity Writers
```

## DrawingPlan

DrawingPlan decides the drawing-level organization:

- Which drawings to generate.
- View placement.
- View scale.
- Sheet size.
- Drawing origin.
- Layer mapping.
- Annotation rules.
- Dimension strategy.
- Title text.

Example:

```json
{
  "drawing_id": "general_arrangement_001",
  "sheet": {
    "size": "A1",
    "scale": 200
  },
  "views": [
    {
      "view_id": "elevation_main",
      "type": "elevation",
      "origin": [0, 0],
      "scale": 200
    },
    {
      "view_id": "plan_main",
      "type": "plan",
      "origin": [0, -120],
      "scale": 200
    }
  ]
}
```

## CAD Instruction Protocol

CAD instructions are deterministic drawing commands, not AutoCAD API code.

Initial instruction types:

- line.
- polyline.
- arc.
- circle.
- text.
- mtext.
- aligned_dimension.
- linear_dimension.
- hatch.
- block_reference.
- leader.

Example:

```json
{
  "kind": "polyline",
  "id": "girder_outline_elevation_001",
  "layer": "BRIDGE_GIRDER",
  "view_id": "elevation_main",
  "source_component_id": "main_girder",
  "points": [
    [0, 0],
    [40, 0],
    [110, 0],
    [150, 0]
  ],
  "closed": false
}
```

## Deterministic Geometry

All geometry should be generated by code from BridgeModel values. Examples:

- Span station positions from cumulative span lengths.
- Pier positions from support lines.
- Elevation view x-coordinates from bridge axis distance.
- Plan view deck edges from centerline offset by deck width.
- Cross-section outlines from section parameters.

## Drawing Conventions

The generator should own engineering drawing conventions:

- Layer names.
- Line weights.
- Line types.
- Text height.
- Dimension style.
- Annotation offsets.
- View spacing.
- Title and note placement.

These should be configurable as templates rather than hardcoded everywhere.
